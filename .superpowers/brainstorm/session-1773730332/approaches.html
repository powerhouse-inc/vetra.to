Vetra UI Overhaul — 3 Approaches

How should we structure this mega-redesign across all 4 workstreams?

A

Design System First, Then Pages

Recommended. Start by aligning globals.css + Tailwind tokens to the design system (colors, typography, spacing, borders). Then rebuild pages on top of the updated foundation.

Order:
1. Design tokens & globals.css alignment
2. Shared components (Navbar, Footer, Cards)
3. Homepage rebuild (from design system landing template + Framer content)
4. Cloud landing page rebuild
5. Cloud management pages UX rethink
6. Builder pages rethink (listing + profiles)
7. Remove Framer rewrites
Pros: Every page benefits from the token work. No rework. Clean dependency chain.
Cons: Slower to see full-page results. Token work isn't flashy.
B

Homepage First, Pull Tokens As Needed

Start with the most visible page — the homepage. Extract tokens from the design system as you go, let the design system alignment emerge organically from building real pages.

Order:
1. Homepage rebuild (tokens extracted inline)
2. Cloud landing page rebuild
3. Consolidate tokens into globals.css
4. Cloud management pages
5. Builder pages
6. Polish pass across all pages
Pros: Fastest to a "wow" moment. Homepage drives token decisions with real context.
Cons: Risk of inconsistency. Needs a consolidation pass later. May duplicate token work.
C

Parallel Tracks with Merge

Split into two parallel tracks: one person does tokens + shared components, another does page layouts. Merge at the end.

Track A: Design tokens → Shared components → Navbar/Footer
Track B: Homepage layout → Cloud landing → Cloud mgmt → Builders
Merge: Apply tokens to Track B pages
Pros: Fastest overall if you have parallelism. Both tracks make progress simultaneously.
Cons: Merge conflicts. Track B may build on wrong assumptions. Coordination overhead.

My Recommendation: Approach A

The design system is comprehensive and well-defined. Building on solid tokens means every page we touch automatically looks right. The Framer content is already captured — we have all the copy, sections, and visual structure. The design system even has a landing page template we can adapt. This is the cleanest path.
