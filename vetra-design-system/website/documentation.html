What is Vetra?

Vetra is a comprehensive development platform that provides a unified interface for building, deploying, and managing modern web applications. It combines a powerful component library with backend services, authentication, and real-time data synchronization.

Whether you are building a simple landing page or a complex enterprise dashboard, Vetra provides the tools and patterns you need to ship with confidence. Our component library follows accessibility best practices and supports full theming out of the box.

✓
Tip If you are new to Vetra, we recommend starting with the Quick Start guide. It walks you through creating your first project in under five minutes and covers the most common patterns you will use every day.

Key Concepts

Before diving into the code, it helps to understand a few core concepts that underpin the Vetra platform. These patterns are used throughout the library and will make the rest of the documentation easier to follow.

Workspaces are top-level containers that hold your projects, team members, and configuration. Each workspace has its own billing plan, permissions model, and environment variables. You can think of a workspace as an organization or company account.

Components are the building blocks of your UI. Vetra ships with over 40 pre-built components that are fully typed, accessible, and themeable. Each component accepts a consistent set of props for sizing, variants, and state management.

Hooks provide reactive data bindings for common operations like fetching data, managing form state, and subscribing to real-time updates. They integrate seamlessly with the component library and follow React conventions.

Installation

Install Vetra using your preferred package manager. We recommend using npm or pnpm for the best compatibility with our toolchain.

Terminal 
# Install core packages
npm install @vetra/core @vetra/components

# Install optional peer dependencies
npm install @vetra/icons @vetra/hooks

Once installed, wrap your application with the VetraProvider component. This sets up the theme context, provides default configuration, and initializes the real-time connection if enabled.

app.tsx 
import { VetraProvider } from '@vetra/core';
import { Button, Card } from '@vetra/components';

export default function App() {
  return (
    <VetraProvider theme="light">
      <Card padding="lg">
        <h1>Hello, Vetra!</h1>
        <Button variant="primary" size="md">
          Get Started
        </Button>
      </Card>
    </VetraProvider>
  );
}

Configuration

Vetra can be configured through a vetra.config.ts file at the root of your project. This file lets you customize the theme, set default component props, configure API endpoints, and enable feature flags.

The configuration file is automatically detected and loaded at build time. All options have sensible defaults, so you only need to specify the values you want to override.

vetra.config.ts 
import { defineConfig } from '@vetra/core';

export default defineConfig({
  theme: {
    primary: '#04C161',
    radius: 8,
    fontFamily: 'Inter, sans-serif',
  },
  api: {
    baseUrl: 'https://api.vetra.io/v3',
    timeout: 10000,
  },
  features: {
    realtime: true,
    analytics: true,
  },
});

Button Component Props

The Button component is one of the most commonly used components in the library. Here is a complete reference of its accepted props.

Prop Type Default Description
variant "primary" | "secondary" | "ghost" | "destructive" "primary" Visual style variant of the button
size "sm" | "md" | "lg" "md" Controls padding and font size
disabled boolean false Disables the button and applies muted styles
loading boolean false Shows a spinner and disables interaction
icon ReactNode undefined Icon element rendered before the button label
onClick (e: MouseEvent) => void undefined Callback fired when the button is clicked
